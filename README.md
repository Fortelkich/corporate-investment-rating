# Прогнозирование кредитного рейтинга компаний

Проект посвящен построению моделей машинного обучения для прогнозирования укрупненных групп кредитного риска компаний на основе финансовых показателей.

## Цель проекта

Цель — разработать ML-пайплайн для прогнозирования кредитного рейтинга компаний и определить финансовые признаки, наиболее связанные с уровнем кредитного риска.

Дополнительно рассматривается proxy-задача: прогнозирование показателя `operatingCashFlowSalesRatio`, который связан с кредитным рейтингом и отражает способность компании генерировать операционный денежный поток относительно выручки.

## Данные

В проекте используются два датасета:

* `corporate_rating.csv` — данные компаний с кредитными рейтингами;
* `sp500_financial_dataset_2010_2025.csv` — расширенный датасет финансовых показателей компаний S&P 500.

Кредитные рейтинги были сгруппированы в несколько классов риска:

* `Low Risk`;
* `Medium Risk`;
* `High Risk`;
* `Highest Risk`.

Редкие классы были удалены, чтобы улучшить устойчивость модели.

## Основные этапы

### 1. Прогнозирование кредитного рейтинга

В первой части решается задача многоклассовой классификации.

Были выполнены:

* предобработка данных;
* группировка рейтингов;
* обработка выбросов;
* масштабирование признаков;
* кодирование категориальных переменных;
* обучение и сравнение нескольких моделей.

Сравнивались модели:

* Logistic Regression;
* Decision Tree;
* Random Forest;
* Gradient Boosting;
* XGBoost;
* SVM;
* KNN;
* Dummy Classifier.

Основной метрикой выбрана `Macro F1`, так как классы распределены неравномерно.

Лучший результат после подбора гиперпараметров показала модель `Gradient Boosting`.

## Результаты классификации

| Метрика           | Значение |
| ----------------- | -------: |
| Accuracy          |   0.6741 |
| Balanced Accuracy |   0.5466 |
| Macro F1          |   0.5596 |
| Weighted F1       |   0.6676 |
| MAE               |   0.3926 |

Наиболее важными признаками оказались показатели, связанные с операционным денежным потоком, особенно:

* `operatingCashFlowPerShare`;
* `operatingCashFlowSalesRatio`.

## 2. Прогнозирование proxy-показателя

Во второй части строится регрессионная модель для прогнозирования:

```text
operatingCashFlowSalesRatio = operating_cash_flow / revenue
```

После расчета целевой переменной признаки `operating_cash_flow` и `revenue` удаляются, чтобы избежать утечки данных.

Сравнивались:

* Dummy Regressor;
* Ridge;
* Random Forest Regressor;
* Gradient Boosting Regressor.

Также проверялись методы уменьшения размерности:

* PCA;
* TruncatedSVD;
* без уменьшения размерности.

Лучший результат показал `RandomForestRegressor` без уменьшения размерности.

## Результаты регрессии

| Метрика | Значение |
| ------- | -------: |
| RMSE    |   1.1459 |
| MAE     |   0.3876 |
| R2      |   0.5769 |

Модель улучшила MAE примерно на 29.3% по сравнению с базовой dummy-моделью.

## Ограничения

* исходный датасет с рейтингами небольшой;
* классы рейтингов несбалансированы;
* модель прогнозирует укрупненные группы риска, а не точные рейтинги;
* `operatingCashFlowSalesRatio` является proxy-показателем, а не полноценной заменой кредитного рейтинга.

## Вывод

В проекте построен ML-пайплайн для прогнозирования кредитного риска компаний. Лучшей моделью классификации стала `Gradient Boosting`, а анализ важности признаков показал значимость показателей операционного денежного потока. Во второй части была построена регрессионная модель для прогнозирования proxy-показателя `operatingCashFlowSalesRatio`.
